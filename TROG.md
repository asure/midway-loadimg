# Trog — /OLD Mode Issues

This document describes the remaining issues with Trog support under the
`/OLD` flag. The core encoding fixes (bpp=8, IMG_STRIDE, global suffix
cache) are working. The `section_base_bit` fix for two `***>` sections
and the `+META` global cache resolved most SAG cascade issues. Remaining
problems: Motorola ASM format (cosmetic, kept as-is), BBB background
encoding cascade, and 2 missing BBB labels in IMGTBL.ASM.

---

1. **BBB handler → IMGTBL.ASM**: TROGMOUTH/MOUTHBACK should be written to the

   BGNDTBL.ASM. These are face images from TROGFACE.BDD.

   sub-module sections. The reference uses a different assignment scheme.

4. **IMGTBL.GLO/BGNDTBL.GLO formatting**: Need Motorola hex format for
   .globl entries, tab-based spacing, and blank line separators.

Format: `/OLD` outputs Motorola hex (`>%X`). Add `/H` for Intel hex (`0%XH`).

## Category 1: `+META` Derived Images (Resolved)

### The Problem (original)

Some `+META` entries referenced a base name that did **not exist** in any IMG
file. Example from `TROG.LOD` line 294:

```
---> BLOOVHAND1+VHAND1L,BLOOVHAND2+VHAND2L
```

- `BLOOVHAND1+VHAND1L` → base name `BLOOVHAND1` → found in `TROGPLAY.IMG`
- `BLOOVHAND2+VHAND2L` → base name `BLOOVHAND2` → **NOT found in any IMG**

### Resolution in v0.95

The `+META` suffix cache was changed from per-line (local) to global
(static), matching LOAD.EXE behavior where all images with the same
suffix share the same SAG across the entire LOD. When `BLOOVHAND2+VHAND2L`
is encountered, the cache finds `+VHAND2L` from `BLOOVHAND1+VHAND1L` and
reuses its SAG. LOAD.EXE also reuses the same SAG (via suffix match),
as confirmed by the reference where both entries have the same SAG value.

The cache also stores the **final TBL SAG value** (`base_addr + relative_offset`)
so that cache hits across different `***>` sections get the correct base
address.

### What `+META` modifiers might mean

Looking at the suffix patterns in Trog:

| Suffix | Pattern | Possible Meaning |
|--------|---------|-----------------|
| `+STP0L` | `*STP0+STP0L`| "Stop frame 0" — share same animation frame |
| `+GHL1L` | `GWEN*+GHL1L` | "Gwen head" — character-specific overlay |
| `+VHAND1L` | `*VHAND1+VHAND1L` | "Vertical hand" — hand position variant |
| `+VHAND2L` | `*VHAND2+VHAND2L` | "Vertical hand 2" — different hand position |
| `+HHAND2L` | `*HHAND2+HHAND2L` | "Horizontal hand 2" |
| `+eggL` | `*egg+eggL` | Egg/object variant |

The pattern `BLOOVHAND2+VHAND2L` suggests:
- Base name `BLOOVHAND2` should be a variant of `BLOOVHAND1`
- The `+VHAND2L` modifier indicates "apply vertical hand position 2"
- Pixel data is probably generated by **translating** `BLOOVHAND1`'s pixels
  by the offset encoded in the suffix

### What LOAD.EXE likely does

1. Look up base name in IMG → if found, use its pixel data directly
2. If NOT found, treat it as a **derived image**:
   - Find the nearest matching base image (strip trailing number/letter)
   - Apply the suffix modifier (offset, flip, palette swap)
   - Generate pixel data and encode it

### Impact on SAG Cascade

When `BLOOVHAND2` is skipped (base not found), it gets:
- No `ImageEntry` (g.n_images not incremented)
- No IRW data
- No TBL entry

The reference has `BLOOVHAND2` at IMGTBL.ASM SAG[62] with 320 bytes of
properly encoded data. Our tool skips it. This:
1. Leaves a 320-byte gap in the IRW (our data is 320 bytes shorter)
2. Shifts all subsequent SAGs by 320 bytes
3. All later images in IMGTBL.ASM have wrong SAGs

### Resolution in v0.95

The BBB compression cascade was fixed by:
1. **Forcing 8bpp for BBB backgrounds in /OLD mode** (was auto-detecting to lower bpp)
2. **No zero-compression in /OLD mode** — `g.zon=0` (LOAD.EXE only outputs uncompressed)
3. **Consistent sizx_a in second pass** — comp_bits calculation now uses sizx_a (not w)
4. **OUT_STRIDE minimum 1** for /OLD mode (was minimum 3, breaking 1-pixel font images)

All 9 per-game TBLs are now byte-exact. The remaining format differences
are in BGNDTBL.ASM, BGNDPAL.ASM, IMGTBL.GLO (BBB handler output), and
the TROGMOUTH/MOUTHBACK entries missing from IMGTBL.ASM.

Note: BLOOVHAND2 and other +META derived images that don't exist in any
IMG file are handled by the global suffix cache (matching LOAD.EXE 4.50).

## Category 2: Motorola ASM Format

### The Problem

Our tool outputs Intel-style hex (`03000000H`) while the ref uses
Motorola-style (`>3000000`). Additionally, the ref separates SAG and PAL
onto separate `.long` lines while our tool combines them:

```asm
; Reference (Motorola, separate lines):
.long	>3005A00
.long	PINK16

; Our output (Intel, combined line):
.long   03005a00H,PINK16
```

### What needs to change

The Motorola format is required by the TMS34010 GSP assembler (`gspa`):

| Feature | Motorola (needed) | Intel (current) |
|---------|-------------------|-----------------|
| Hex prefix | `>3000000` | `03000000H` |
| Long syntax | `.long` | `.long` |
| SAG/PAL | Separate lines | Combined on one line |
| File headers | Has `.FILE`, `.OPTION`, `.include`, `.even` | Just `.DATA` |

### Motorola format (deferred)

The Motorola ASM format (hex prefix `>`, separate SAG/PAL lines, file
headers) is cosmetic — the Intel format (`0...H`, combined SAG+PAL) works
correctly with the assembler. Not required for functional correctness.
Kept as-is unless a specific toolchain requires it.

## Notes

- Labels whose base name doesn't exist in any `.IMG` file are **derived
  images**. The global +META suffix cache now shares their SAG with the
  first matching suffix entry (matching LOAD.EXE behavior).
- The remaining IMGTBL.ASM SAG cascade at [232] is from BBB background
  encoding differences, not from missing derived images.

## Current SAG Match Status

```bash
# Check IMGTBL.ASM SAG match count
cd worktrog
python3 << 'EOF'
import re
def get_sags(lines):
    sags = []
    for l in lines:
        l = l.strip()
        m = re.search(r'\.long\s+>([0-9A-Fa-f]+)', l)
        if not m:
            m = re.search(r'\.long\s+0([0-9A-Fa-f]+)H', l)
        if m:
            sags.append(int(m.group(1), 16))
    return sags

ref = get_sags(open('../reftrog/IMGTBL.ASM').readlines())
our = get_sags(open('IMGTBL.ASM').readlines())
match = sum(1 for r, o in zip(ref, our) if r == o)
print(f'IMGTBL.ASM: {match}/{len(our)} SAG match (ref has {len(ref)})')
if match < len(our):
    for i, (r, o) in enumerate(zip(ref, our)):
        if r != o:
            base = 'SAME' if (r & 0xFF000000) == (o & 0xFF000000) else f'BASE:0x{r&0xFF000000:X}->0x{o&0xFF000000:X}'
            off = 'SAME' if (r & 0xFFFFFF) == (o & 0xFFFFFF) else f'OFF:0x{r&0xFFFFFF:X}->0x{o&0xFFFFFF:X}'
            print(f'  First cascade at [{i}]: ref=0x{r:X} our=0x{o:X}  {base} {off}')
            break
EOF
```
